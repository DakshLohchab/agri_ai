    \begin{itemize}[leftmargin=*, label=\textcolor{brandblue}{\textnormal{\faCheckCircle}}, itemsep=3pt]

        \item \textbf{Multilingual Normalization:} Translates regional queries into a unified base language for the AI reasoning engine, then accurately localizes the generated response back to the farmer.
        \item \textbf{Resilient Hybrid Routing:} Dynamically utilizes a powerful \textbf{Express \& LangGraph cloud pipeline} when connected, but gracefully degrades to \textbf{local Ollama inference models} (Llama3, Mistral) in zero-network zones to guarantee zero downtime.
        \item \textbf{Context-Aware Memory:} The app stores conversation context locally, allowing the AI to remember the farmer's specific crop cycle, soil type, and past issues without requiring repeated explanations.
        \item \textbf{Live Integrations:} Enriches answers with real-time Open-Meteo weather forecasting and live AGMARKNET Mandi prices, making the AI's advice highly actionable and economically relevant.

{\Large \textnormal{\faExchange}\ \bfseries Runtime Request and Response Lifecycle}

\vspace{3mm}
\textbf{Execution model overview.}
Every user query initiates a resilient lifecycle designed to guarantee a localized, domain-accurate response. The system first captures input via text or native voice APIs, immediately normalizing the request into an English base for unified processing. Based on backend health and query intent, the orchestrator dynamically routes the request to either the primary LangGraph cloud pipeline or the local client fallback reasoning engine. After enriching the context with live external data, the system synthesizes a structured markdown response. Finally, this payload is translated back to the user's selected language, persisted to local storage alongside the base response, and delivered through the UI and TTS engines.
